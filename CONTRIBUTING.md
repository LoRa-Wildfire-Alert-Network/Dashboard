# Contributing

Thanks for taking the time to contribute! Please follow these guidelines to keep things smooth for everyone.

## Getting Started

1. Fork the repository
2. Clone your fork:
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Push to your fork and open a Pull Request

## Code Style

- Keep code clean and readable
- Comment where necessary
- Follow the existing conventions in the codebase

## Pull Requests

- One feature or fix per PR
- Write a clear description of what your PR does and why
- Reference any related issues (e.g. `Closes #42`)
- Make sure existing tests pass before submitting

## Reporting Bugs

Open an issue and include:
- A clear description of the problem
- Steps to reproduce it
- Expected vs actual behavior
- Your environment (OS, version, etc.)

## Reporting Vulnerabilities

Please do **not** open a public issue for security vulnerabilities. Instead, email [admin@lorawildfirealertnet.work](mailto:admin@lorawildfirealertnet.work) directly. You can expect an initial response within 48 hours. If the vulnerability is confirmed, we will work on a fix and credit you in the release notes.

## Questions

Open a discussion or issue and we'll get back to you.
